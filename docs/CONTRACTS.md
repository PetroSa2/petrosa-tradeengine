# Petrosa Trade Engine Contracts

## Signal Ingestion Contract

### NATS Subscription
The Trade Engine subscribes to NATS to receive trading signals generated by the analytical bots (e.g., `ta-bot`, `realtime-strategies`). 
The subscription subject is defined by the `NATS_SIGNAL_SUBJECT` environment variable.

**Critical Requirement:** To ensure the engine successfully receives all strategy-specific signals, this variable MUST be configured as a wildcard (e.g., `signals.trading.*`). 
If configured as a literal string without a wildcard (e.g., `signals.trading`), the engine will silently ignore all signals published to sub-topics like `signals.trading.rsi_reversal`.

### Signal Payload Schema
The engine expects incoming NATS messages to be JSON-encoded representations of the `Signal` Pydantic model (`contracts/signal.py`). 

A valid organic trade signal must include the following mandatory fields at a minimum:
*   `strategy_id` (str): Unique identifier for the strategy.
*   `symbol` (str): Trading pair (e.g., "BTCUSDT").
*   `action` (str): Desired action ("buy" or "sell").
*   `price` (float): The current or target execution price.
*   `quantity` (float): The base asset quantity to trade (or 0.0 to rely on the engine's dynamic position sizing).
*   `current_price` (float): The current market price of the asset.
*   `confidence` (float): Strategy confidence level [0.0 - 1.0].
*   `source` (str): Originating service (e.g., "ta_bot").
*   `strategy` (str): Human-readable name of the strategy.

Failure to provide all required fields will result in a Pydantic validation error, and the signal will be discarded.
